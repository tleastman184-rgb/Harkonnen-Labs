# PDF Standards

Use this skill when the repo lives in a standards-heavy or regulated context and
the pack needs to read, cite, compare, or extract requirements from PDF-based
standards or compliance documents.

## Working Rules

- Treat standards PDFs as evidence, not folklore.
- Quote sparingly and prefer concise paraphrases plus section/page references.
- Distinguish mandatory language ("must", "shall", "required") from guidance
  language ("should", "recommended", "may").
- If multiple documents disagree, surface the conflict explicitly instead of
  blending them.
- Record which document version or revision was consulted.

## Good Practice

- Capture the exact clause, section, or appendix that drives an implementation
  choice.
- Translate standards requirements into repo-local checks, constraints, or
  evidence artifacts.
- Preserve auditability: when a change is justified by a standard, note the
  document, section, and reasoning chain.

## Avoid

- Pretending a remembered summary is equivalent to reading the standard.
- Treating vendor guidance as if it were the governing standard.
- Over-quoting long passages when a precise paraphrase and citation would do.
